## 3. Five Moats

### Moat 1: Local-First Speed Architecture
- **Claim:** Sub-300ms end-to-end latency with local Whisper (tiny/base) + deterministic refinement
- **Why it matters:** Wispr Flow is ~800ms. SuperWhisper is fast but Mac-only. We can be faster AND cross-platform via WebAssembly + WebGPU.
- **Sustainability:** High. Requires deep optimisation of audio pipeline, model quantisation, and inference scheduling.

### Moat 2: Developer Workflow Integration
- **Claim:** Native understanding of code syntax, file references, commit messages, PR comments, terminal commands
- **Why it matters:** Developers are high-value users (Cursor, VS Code, Windsurf). Wispr Flow supports this but not as deeply as a developer-native tool.
- **Sustainability:** Medium-High. Built through community contributions and IDE-specific adapters.

### Moat 3: Team Dictionary Network Effects
- **Claim:** Shared dictionaries, snippets, and style profiles that improve with every team member
- **Why it matters:** The more teammates use it, the better the transcriptions for everyone. Creates lock-in.
- **Sustainability:** High. Classic network effect moat.

### Moat 4: BYOK + Custom Model Routing
- **Claim:** Users bring their own API keys AND can route through self-hosted models, on-prem gateways, or private LLMs
- **Why it matters:** Enterprises won't adopt without data sovereignty. Wispr Flow is cloud-only.
- **Sustainability:** Medium. Technical differentiation that requires ongoing protocol maintenance.

### Moat 5: Compliance-First Positioning
- **Claim:** HIPAA-ready on free tier, SOC 2 Type II path, zero-data-retention architecture
- **Why it matters:** Healthcare, legal, finance are high-ARPU verticals. Wispr Flow charges enterprise prices for compliance we can offer earlier.
- **Sustainability:** High. Compliance is a barrier to entry for new competitors.

## 4. Name Analysis: OpenFlow vs OpenFlowKit

### Trademark Risk
- **USPTO Registration:** OPENFLOW (Serial 85319202, Reg #4795420) — LIVE and RENEWED
- **Owner:** Open Networking Foundation (networking protocol for SDN)
- **Classes:** 9 (computer software) and 42 (design/development services)
- **Risk:** **MODERATE-HIGH**. Same class (software), but different industry. However, both are software products. Likelihood of confusion could be argued if we gain traction.

### GitHub Conflicts
- At least **5 repos** already use "OpenFlow" for voice dictation, creating brand dilution.

### Recommendation
- **Short term:** Keep "OpenFlowKit" (the "Kit" suffix differentiates sufficiently for now)
- **Medium term:** Register trademark for "OpenFlowKit" in Class 9 for voice/speech software
- **Alternative names to secure:** FlowKit, VoxKit, SpeakKit, AuralKit, DictateKit, WhisperKit (taken by Apple)
- **Best alternative:** **FlowKit** — short, memorable, no trademark conflicts found, available domains likely

## 5. Voice-Training Approach: How to Beat Wispr Flow

### Current Benchmarks
- Wispr Flow: 220 WPM, ~800ms latency, 90% zero-edit accuracy, 97.2% raw accuracy
- Whisper Large-v3: 2.7% WER (clean), 8-12% WER (real-world)
- faster-whisper: 4x speedup over vanilla Whisper

### Our Target: 250+ WPM, <300ms latency, 95%+ zero-edit accuracy

### Technical Strategy

1. **Hybrid STT Architecture**
   - **Tier 1:** Web Speech API (browser built-in) for instant first-pass transcription (<100ms)
   - **Tier 2:** Local whisper.cpp (tiny/base) via WASM/WebGPU for offline accuracy
   - **Tier 3:** Cloud fallback (Groq/Deepgram) for complex audio or non-English
   - **Result:** Always-on speed with accuracy fallback

2. **Streaming Refinement Pipeline**
   - Process transcription in 500ms chunks, not full utterances
   - Apply deterministic refinement (fillers, backtracks) in real-time
   - Run LLM polish only on final pause (>800ms silence)
   - **Result:** Feels instant, not batch-processed

3. **Context-Aware Personalisation**
   - Screen context capture (with explicit consent) to bias vocabulary
   - App-specific formatting rules (Slack = concise, Email = formal, Code = syntax-aware)
   - Personal dictionary with phonetic matching for names/technical terms
   - **Result:** Higher zero-edit rate than generic transcription

4. **Voice Training / Adaptation Layer**
   - On-device fine-tuning of Whisper on user's corrected transcripts (federated, privacy-preserving)
   - Accent adaptation through 30-second calibration samples
   - Vocabulary injection via pronunciation guides ("My name is spelled K-O-T-H-A-R-I")
   - **Result:** Gets better the more you use it — unlike cloud-only competitors

5. **Self-Correction Intelligence**
   - Detect spoken self-corrections ("no wait, actually...") and apply them in real-time
   - Use prosody (pitch/speed changes) to detect hesitation vs emphasis
   - Maintain a 3-second rolling buffer to allow retroactive edits
   - **Result:** 95%+ zero-edit accuracy target

6. **Developer-Specific Optimisations**
   - CamelCase/snake_case/kebab-case detection and formatting
   - File path and symbol name matching against open workspace
   - Terminal command validation and suggestion
   - Code comment vs code body context switching
   - **Result:** Developer NPS > 50

## 6. Browser MVP Implementation

### What Ships Now

1. **VoiceCapture component** (`src/components/VoiceCapture.tsx`)
   - Web Speech API integration with `SpeechRecognition`
   - Push-to-talk or continuous mode
   - Real-time transcript display
   - Visual feedback (recording state, confidence meter)

2. **TerminalBridge component** (`src/components/TerminalBridge.tsx`)
   - WebSocket connection to local relay (for desktop injection)
   - Copy-to-clipboard fallback
   - Command-mode interface ("make it formal", "bullet points")

3. **Dictation pipeline** (`src/core/dictation.ts`)
   - Orchestrates: Capture → Transcribe → Refine → Inject
   - DictationResult schema per AGENTS.md
   - Latency tracking per stage

4. **Enhanced refinement** (`src/core/refine.ts`)
   - Filler word removal
   - Self-correction resolution
   - Developer formatting (camelCase, snake_case, newlines)
   - Snippet expansion
   - Terminal punctuation

5. **Tests** (`src/core/*.test.ts`)
   - Refinement logic tests
   - Dictation pipeline tests
   - Capture state machine tests
